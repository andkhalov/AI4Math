## 3. Проверка новых возможностей

### 3.1 Artifact cache

В stage 1 агент вызвал `pdf_read` × 3 → все вернули preview + artifact_id. Затем агент **самостоятельно вызвал** `load_artifact("pdf_read_0001")` × 3 когда понял что для IEEE-цитаты нужен полный текст.

```
▸ pdf_read ai4math-mcp     → preview + [artifact: pdf_read_0001]
▸ load_artifact ai4math-mcp → полный текст
(повторилось 3 раза)
```

**Паттерн работает как задумано:** полные 5-8 KB PDF-outputов не висят в истории для rest of session — агент подгружает их точечно.

### 3.2 Token budget reporting

Агент сам вызвал `token_budget()` в stage 3 и вывел в конец ответа:

> `(Бюджет: 934k / 2M — 46%, остаток 1.06M)`

Это ровно то поведение, которое мы заложили в recipe — **пользователь видит расход в моменте без доп. действий**.

### 3.3 Announce-before-action

Комментарии появились в stage 2+ в таком виде:

> **Шаг 1 ✓** — создан `experiment/simulate.py` для симуляции Бернулли(1/2) усреднений. Использованы параметры n = 10, 100, 1000, 10000 и t = 0.05, 0.1, 0.2. M = 100 000 испытаний на пару (n, t), seed = 42 для воспроизводимости.

Агент проговаривает что делает, не просто молча вызывает тулы. Перед каждым tool call — текстовая префраза.

### 3.4 Goose context display

Banner правильно показывает `auto-compact при 102k` для DeepSeek (128k × 0.78 ≈ 100k). Синхронно с `GOOSE_AUTO_COMPACT_THRESHOLD`.

## 4. Проблемы и фиксы по ходу

1. **smart_approve в `run` mode → deadlock.** Первая попытка stage 1 упала с `Error: not connected` на shell-команде. Причина: smart_approve ожидает одобрения от пользователя, но в non-interactive `run` mode одобрять некому. **Фикс (уже в коде):** дефолт теперь `auto` для `run` mode, `smart_approve` только для interactive session.

2. **DeepSeek иногда пропускает namespace** (`-32002: Tool 'pdf_read' not found` → через ход `pdf_read ai4math-mcp`). Поведение модели, не баг. Агент сам восстанавливается через инструкции recipe про `-32002` ошибки.

3. **Автоматическая инициализация git.** Агент сделал `git init` когда обнаружил что папка не репозиторий. Корректное поведение — не было запрета в AGENT.md.

---

## 5. Качество результата

**Lean-утверждение** (`chebyshev_inequality_n100_t0p1`) компилируется в Mathlib 4.26 через `norm_num`. Хотя это арифметическое тождество (`1/(4*100*0.01) = 1/4`), а не полная формализация вероятностной оценки — это соответствует тому что просила задача: «хотя бы одна конкретная оценка с n = 100, t = 0.1 типа Pr(…) ≤ 1/4».

**Эксперимент** воспроизводим (seed=42), результаты сходятся с теорией: эмпирические вероятности резко ниже границы Чебышева при больших n, Хёффдинг ложится к ним значительно плотнее. Графики читаемые (log-scale).

**REPORT.md** содержит все 5 разделов, IEEE-цитаты из `lit.md`, и деформализацию Lean-утверждения. Размер — 7.9 KB, плотный.
